## Task 3: Reduce profile smoothing

**File:** `scripts/build-zwift-data.mjs` — `smoothProfile()` function (~line 798–803)

- [ ] **Step 1: Change smoothing parameters**

Find this block:
```js
function smoothProfile(rawProfile, totalRouteKm) {
  if (!Array.isArray(rawProfile) || rawProfile.length < 5) return rawProfile ?? [];

  const windowKm = Math.min(Math.max((Number(totalRouteKm) || 0) * 0.024, 0.28), 0.68);
  const firstPass = smoothProfilePass(rawProfile, windowKm);
  return smoothProfilePass(firstPass, Math.max(windowKm * 0.82, 0.24));
}
```

Change to:
```js
function smoothProfile(rawProfile, totalRouteKm) {
  if (!Array.isArray(rawProfile) || rawProfile.length < 5) return rawProfile ?? [];

  const windowKm = Math.min(Math.max((Number(totalRouteKm) || 0) * 0.016, 0.28), 0.68);
  const firstPass = smoothProfilePass(rawProfile, windowKm);
  return smoothProfilePass(firstPass, Math.max(windowKm * 0.70, 0.24));
}
```

- [ ] **Step 1: Load app on physical device**

With `npm run serve` running, open the app on your mobile device. Use any mock scenario so you don't need Xert credentials.

- [ ] **Step 2: Audit these areas in order**

Go through each area and note any layout issues (overflow, clipping, cramped controls, illegible text, tap targets too small):

1. **Auth screen** — login form, dev panel data source selector
2. **Header** — title + refresh/sign-out buttons
3. **Status card** — freshness badge, FTP/weight/W/kg pills, bucket bars
4. **Recommendation section** — rec title, subtitle, WOTD banner if visible
5. **Time controls** — time slider + label, timing toggle buttons, avg speed row
6. **Route cards** — full recommendation cards, compact cards in "more time" / "other"
7. **World filter row** — today-only toggle, guest worlds picker
8. **Route Inspector** — search input, world/bucket filter dropdowns, route picker
9. **Settings section** — username/password fields, unit toggle, data source selector

- [ ] **Step 3: Fix any issues found**
